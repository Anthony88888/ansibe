1.4.2 ansible playbook 使用条件判断 when ( * )

1. 编写一个 DBserver.yml 的 playbook 实现根据不同的 CentOS 版本，安装不同的数据库软件
[root@ansible playbook]# vi /etc/ansible/hosts 
[root@ansible playbook]# 
[root@ansible playbook]# cat /etc/ansible/hosts 
[nodes]
192.168.7.113
[root@ansible playbook]#
[root@ansible playbook]# 
[root@ansible playbook]# 
[root@ansible playbook]# cat DBserver.yml 
---
- hosts: nodes
  remote_user: root

  tasks:
    - name: install DBServer software in centos6
      yum: name=mysql-server
      when: ansible_distribution_major_version=="6"

    - name: install DBserver software in centos7
      yum: name=mariadb-server
      when: ansible_distribution_major_version=="7"

1.4.3 ansible playbook 使用循环 with_items ( * )

1. 我们使用循环结构多次执行同一个 task 任务
[root@ansible playbook]# 
[root@ansible playbook]# cat /etc/ansible/hosts 
[nodes]
192.168.7.113
[root@ansible playbook]# 
[root@ansible playbook]# 
[root@ansible playbook]# vi adduser.yml 
[root@ansible playbook]# 
[root@ansible playbook]# cat adduser.yml 
- hosts: nodes
  remote_user: root
  tasks:
    - name: add some group
      group: name={{item}} state=present
      with_items:
        - group1
        - group2
        - group3

    - name: add some user
      user: name={{item.username}} group={{item.usergroup}} state=present
      with_items:
        - {username: 'user1', usergroup: 'group1' }
        - {username: 'user2', usergroup: 'group2' }
        - {username: 'user3', usergroup: 'group3' }
[root@ansible playbook]# 
[root@ansible playbook]# 
[root@ansible playbook]#
[root@ansible playbook]# ansible-playbook adduser.yml

1.4.4 ansible playbook 使用 templates 模板 ( * )

1. templates 可以使用 ansible playbook 中定义的变量，用以生成指定的文件，下面我们以 httpd-install.yml 演示一下
[root@ansible playbook]# cat /etc/ansible/hosts

[root@ansible playbook]# 
[root@ansible playbook]# 
[root@ansible playbook]# cat  httpd-install.yml
---
- hosts: all
  remote_user: root
  
  tasks:
    - name: install httpd 
      yum: name=httpd

    - name: config httpd serivce
      template: src=httpd.conf.j2 dest=/etc/httpd/conf/httpd.conf
      notify: restart httpd service

    - name: copy default index.html
      copy: src=/data/playbook/index.html dest=/usr/share/httpd/noindex/index.html

    - name: manage httpd service
      service: name=httpd state=started enabled=yes

  handlers:
    - name: restart httpd service
      service: name=httpd state=restarted

[root@ansible playbook]# cat httpd-install.yml 
---
- hosts: all
  remote_user: root
  vars:
    httpd_vhosts:
      - web01:
        listen: 8001
        documentroot: '/data/web01'

      - web02:
        listen: 8002
        documentroot: '/data/web02'

  tasks:
    - name: install httpd 
      yum: name=httpd

    - name: config httpd serivce
      template: src=vhostport.conf.j2 dest=/etc/httpd/conf.d/vhostport.conf
      notify: restart httpd service

    - name: copy default index.html
      copy: src=/data/playbook/index.html dest=/usr/share/httpd/noindex/index.html

    - name: manage httpd service
      service: name=httpd state=started enabled=yes
